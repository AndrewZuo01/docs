# createGroup

## 功能介绍

:::info 说明

（1）指定群管理员和群普通成员创建群组；  
（2）群成员建议一次最大 1000 个，因为数量过多可能会导致数据包太大被后台拒绝。

:::

:::caution 注意

（1）如果指定 groupID，则 groupID 不能重复；  
（2）如果不指定 groupID，服务端会生成唯一的 groupID  
（3）所有群成员会收到 OnJoinedGroupAdded 和 OnGroupMemberAdded 回调。

:::

### 函数原型

```ts showLineNumbers
IMSDK.asyncApi('createGroup', operationID: string, groupBaseInfo: Partial<GroupInfo>, memberList:MemberWithCreate[]): Promise<void>
```

### 输入参数

| 参数名称      | 参数类型                                                        | 是否必填 | 描述                                                    |
| ------------- | --------------------------------------------------------------- | -------- | ------------------------------------------------------- |
| operationID   | string                                                          | 是       | 操作 ID，用于定位问题，保持唯一，建议用当前时间和随机数 |
| groupBaseInfo | Partial<[GroupInfo](docs/sdks/Class/Group/GroupInfo.mdx)>       | 是       | 群聊基础信息                                            |
| memberList    | [MemberWithCreate](docs/sdks/Class/Group/GroupMemberInfo.mdx)[] | 是       | 邀请进入的群成员列表                                    |

### 返回结果

> 通过`openim-uniapp-polyfill`包将函数 Promise 化，调用时需要使用`then`和`catch`判断并处理成功和失败回调。

| 参数名称        | 参数类型                                                         | 描述         |
| --------------- | ---------------------------------------------------------------- | ------------ |
| Promise.then()  | Promise<void\>                                                   | 调用成功回调 |
| Promise.catch() | Promise<[CatchResponse](docs/sdks/Class/Response/Response.mdx)\> | 调用失败回调 |

### 代码示例

```js showLineNumbers
import IMSDK from 'openim-uniapp-polyfill';

const groupBaseInfo = {
  groupName: '',
  groupType: 2,
};
const memberList = [
  {
    userID: '',
    roleLevel: 1,
  },
];
IMSDK.asyncApi('createGroup', Date.now(), groupBaseInfo, memberList)
  .then(() => {
    // 调用成功
  })
  .catch(({ errCode, errMsg }) => {
    // 调用失败
  });
```
